# Session Start Checklist

> Purpose: Ensure every engineering session begins with the correct understanding, context, and preparation.

---

## 1. Understand the Objective

- [ ] Read the task description.
- [ ] Identify the business objective.
- [ ] Identify the expected outcome.
- [ ] Confirm the task is fully understood.
- [ ] Clarify any uncertainties before proceeding.

---

## 2. Review Project Documentation

- [ ] Engineering Handbook
- [ ] Enterprise Architecture
- [ ] Engineering Execution Plan
- [ ] Current Milestone
- [ ] Relevant ADRs
- [ ] Decision Log
- [ ] Module README (if applicable)

---

## 3. Audit the Existing System

- [ ] Locate affected modules.
- [ ] Review existing implementation.
- [ ] Identify reusable components.
- [ ] Identify related services.
- [ ] Identify related schemas.
- [ ] Identify related APIs.
- [ ] Identify dependencies.

---

## 4. Scope Validation

- [ ] Define what IS included.
- [ ] Define what IS NOT included.
- [ ] Confirm no scope creep.

---

## 5. Risks

- [ ] Identify breaking changes.
- [ ] Identify migration requirements.
- [ ] Identify documentation updates.
- [ ] Identify testing requirements.

---

## Ready to Begin

- [ ] Everything understood.
- [ ] Architecture reviewed.
- [ ] Scope approved.
- [ ] Ready for implementation.
