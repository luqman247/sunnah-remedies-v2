# AI Quality Standard

## Purpose

Define the minimum quality expected from AI-generated engineering work.

---

# Quality Objectives

AI outputs should be:

- Accurate
- Complete
- Maintainable
- Secure
- Accessible
- Performant
- Well documented
- Production ready

---

# Engineering Quality Gates

Every implementation should pass:

✓ Build

✓ Type Check

✓ Lint

✓ Tests

✓ Browser Verification

✓ Accessibility Review

✓ Performance Review

✓ Security Review

✓ Documentation Review

---

# AI Should Never

- Invent requirements.
- Skip verification.
- Ignore architecture.
- Hardcode editable content.
- Expand scope.
- Leave undocumented changes.

---

# AI Should Always

- Explain assumptions.
- Highlight risks.
- Suggest improvements.
- Update documentation.
- Produce verification reports.

---

# Definition of Quality

Quality means:

Correctness

Consistency

Maintainability

Readability

Performance

Security

Accessibility

Search Visibility

Documentation

---

# Quality Checklist

Before reporting completion:

✓ Acceptance criteria met

✓ Engineering Handbook followed

✓ Architecture respected

✓ Repository standards followed

✓ Documentation updated

✓ Verification complete

---

# Principles Summary

1. Correctness comes before speed.
2. Verification is mandatory.
3. Documentation is part of quality.
4. AI should optimise for long-term maintainability.
5. High-quality engineering is deliberate and repeatable.
