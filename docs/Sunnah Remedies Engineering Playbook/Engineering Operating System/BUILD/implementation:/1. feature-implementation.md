# Feature Implementation Prompt

## Purpose

Implement an approved feature according to the Engineering Handbook, Enterprise Architecture, and current Engineering Execution Plan.

---

## When to Use

Use this prompt whenever implementing a new feature or extending an existing one.

---

## Inputs Required

- Engineering Handbook

- Enterprise Architecture

- Engineering Execution Plan

- Relevant ADRs

- Relevant source files

- Task description

- Acceptance criteria

---

## AI Responsibilities

The AI must:

- Read all relevant documentation.

- Audit the existing implementation.

- Stay within the approved scope.

- Follow Coding Standards.

- Follow Repository Structure.

- Update documentation where required.

- Verify all changes before completion.

---

## Constraints

The AI must not:

- Change architecture.

- Expand scope.

- Invent requirements.

- Hardcode editable content.

- Skip verification.

---

## Expected Deliverables

- Production-ready implementation

- Updated documentation

- Verification report

- Suggested commit message

- Suggested next task

---

## Success Criteria

- Feature implemented.

- Build passes.

- Type-check passes.

- Lint passes.

- UI verified.

- Documentation updated.
