# Example — Good Feature Specification

Feature

Product Comparison

---

Business Goal

Allow users to compare products before purchasing.

---

Acceptance Criteria

- Compare up to four products.

- Responsive.

- Accessible.

- Server-rendered.

- Content editable in Sanity.

---

Technical Notes

Use ProductService.

Reuse ProductCard.

No duplicated components.

---

Why This Is Good

Requirements are measurable.

Scope is defined.

Dependencies identified.
