# Bad Example — Products API

Endpoint

GET /api/products

---

## Problems

Returns every product.

No pagination.

No filtering.

No validation.

No error handling.

No documentation.

No authentication.

No rate limiting.

Exposes internal IDs.

Returns inconsistent responses.

---

## Engineering Problems

Poor scalability.

Security risks.

Slow responses.

Hard to maintain.

Impossible to extend.

---

## Better Solution

Add:

Pagination

Filtering

Validation

Authentication

Documentation

Consistent response format

Logging

Rate limiting
