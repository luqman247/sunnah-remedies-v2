# Release Workflow

> Role: Engineering Release Management

Coordinate the transition from completed engineering work to a production release.

---

# Purpose

Ensure every release is verified, documented, approved, and traceable.

---

# Workflow

Milestone Complete

↓

Verification Complete

↓

Documentation Updated

↓

Release Notes

↓

Git Tag

↓

Founder Approval

↓

Deploy

↓

Smoke Tests

↓

Production Verification

↓

Release Published

---

# Required Inputs

- Verification Report

- Release Notes

- ADRs

- Changelog

- Git Tag

---

# Deliverables

- Production Release

- Changelog

- Release Notes

- Deployment Record

---

# Approval Gates

Engineering

↓

Founder

↓

Production

---

# Post Release

Review:

- Performance

- Errors

- Analytics

- User Feedback

---

# Success Criteria

✓ Production stable

✓ Documentation complete

✓ Release archived
