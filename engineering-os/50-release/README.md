# 50 — Release

> Shipping, versioning, and the durable record of what changed.

The release stage turns verified work into a production change and a permanent record.

## Contents and pointers

- Release workflow: `../90-reference/workflows/release-workflow.md`
- Deployment workflow: `../90-reference/workflows/deployment-workflow.md`

- Release notes template: `../10-design/templates/release-notes.md`
- Worked example: `../30-build/examples/good/release-notes.md`

## Release rules

- No release without a passing verification report.
- Every release produces release notes for two audiences: editorial and engineering.
- Every release records its rollback path.
- The OS itself is versioned in `../CHANGELOG.md`.
